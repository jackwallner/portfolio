Kalshi BTC Bot

An unattended trading bot for Kalshi's hourly Bitcoin "price-above" markets. Capped at $1 of risk per fifteen-minute run.

Source private — happy to walk through Request a walkthrough ↗

§ The pitch

Kalshi runs hourly event-contract markets on whether Bitcoin will close above a given price. The strategy is simple enough to write in one sentence: when the market is greedy, fade the upside; when it's fearful, fade the downside; otherwise, sit out. The whole bot is the operational scaffolding around that one sentence — the part that turns "good idea" into "runs unattended on a cron and doesn't blow up."

§ Specs

LanguagePython 3.11+
AuthRSA-PSS request signing with Kalshi API Key ID + private key
ScheduleGitHub Actions cron · every 15 minutes
Risk cap$1 per run · hard limit, enforced in code
Signal sourceAlternative.me Fear & Greed Index (free, public)
LoggingStructured trade log to trades.log, committed back to repo

§ The decision logic

  1. Find the market. Pull the next hour's BTC price-above contract.
  2. Read the index. Fetch today's Fear & Greed value (0–100).
  3. Pick a side.
    • Index > 60 (greed) → buy YES (price will go above).
    • Index < 40 (fear) → buy NO (price will not).
    • 40–60 (neutral) → no trade.
  4. Cap and place. Size to ≤ $1 total risk for this 15-minute window. Submit. Wait for fill.
  5. Log everything. Decision, fill, error path, hour rollover. Every run leaves a structured row.

The hard part of an algorithmic strategy isn't the algorithm. It's the part where you don't accidentally trade $400 because of a timezone bug.
